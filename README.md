# 🌤 理予的天气 — AstrBot 天气插件

> 基于 **Open-Meteo 免费 API** 的天气查询插件，支持实时天气、3 天预报、24 小时逐时温度曲线、极端天气预警、定时推送以及**自然语言意图识别**。

---

## 📡 数据来源 & API 说明

本插件使用 [Open-Meteo](https://open-meteo.com/) 提供的**完全免费**天气 API，**无需 API Key**。

| 接口 | 用途 | 文档 |
|---|---|---|
| **地理编码 API** (`geocoding-api.open-meteo.com`) | 将地点名称解析为经纬度 | [Geocoding API](https://open-meteo.com/en/docs/geocoding-api) |
| **天气预测 API** (`api.open-meteo.com`) | 获取实时天气、逐日/逐时预报 | [Weather API](https://open-meteo.com/en/docs) |

### 免费额度说明

- ✅ **无 API Key**：直接调用，无需注册
- ✅ **无调用次数限制**（合理频率即可）
- ✅ **非商用免费**：个人项目、开源项目均可自由使用
- ✅ **数据来源**：欧洲中期天气预报中心（ECMWF）等全球气象模型，精度与商业 API 相当

### 注意

- 地理编码优先返回**中国地区**的结果
- 免费 API 数据约 **15–30 分钟**更新一次，非实时秒级更新
- 极端天气预警基于 WMO 天气码判断，非官方气象局预警

## ✨ 功能特性

| 功能 | 说明 |
|---|---|
| **📍 实时天气** | 温度、体感温度、湿度、风速、降水、当前天气描述 |
| **📅 3 天预报** | 每天最高/低温、天气状况、降水概率、降水总量 |
| **⏰ 24 小时逐时** | 逐小时温度变化、降雨概率、带伞建议 |
| **🌂 智能带伞提醒** | 降雨概率 ≥ 50% 自动提示 |
| **⚠️ 极端预警** | WMO 极端天气码自动检测（雷暴、冰雹、大雪等） |
| **📈 温度曲线图** | 24 小时温度折线图，附天气图标 |
| **⏰ 定时推送** | 每日定时向指定会话推送天气简报（需配置） |
| **💬 自然语言触发** | 用户消息含"天气""下雨""温度"等关键词时自动回复（需开启） |
| **🗺 智能地理编码** | 支持 "番禺"、"广州番禺"、"北京" 等多种输入，自动 fallback |

## 📥 安装

### 方式一：通过 AstrBot 管理面板

1. 进入「插件管理」
2. 点击「安装插件」→「手动安装」
3. 填入本插件 GitHub 仓库地址（或直接上传文件夹）

### 方式二：手动放置

将 `astrbot_plugin_tq` 文件夹放入 AstrBot 的 `data/plugins/` 目录，然后重载插件：

```
/plugin reload
```

## ⚙️ 配置说明

### 配置界面（_conf_schema.json）

插件在 AstrBot 管理面板中提供可视化配置，主要分为两个配置段：

#### 💬 自然语言天气识别

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `nlp_weather_enabled` | bool | `false` | 开启自然语言天气触发 |
| `nlp_weather_fallback` | string | `"广州"` | 地点提取失败时的默认查询城市 |

#### ⏰ 天气提醒推送

| 配置项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `reminder_enabled` | bool | `false` | 开启每日定时推送 |
| `reminder_cron` | string | `"0 7 * * *"` | cron 表达式，默认每天早上 7 点 |
| `reminder_targets` | list | `[]` | 推送目标 UMO 列表 |
| `reminder_location` | string | `"番禺"` | 推送时查询的地点 |

### 获取推送目标 UMO

在需要接收推送的群里发送 `/sid`，获取到 `unified_msg_origin` 后填入 `reminder_targets` 列表。

## 🎮 使用方法

### 命令

| 命令 | 示例 | 说明 |
|---|---|---|
| `/天气 [地点]` | `/天气 北京` | 查询指定地点的天气 |
| `/天气` | `/天气` | 查询默认地点（配置中的 default_location）的天气 |
| `/天气状态` | `/天气状态` | 查看插件运行状态、统计数据 |

### 💬 自然语言触发（需开启）

开启 `nlp_weather_enabled` 后，Bot 会自动检测消息中的天气意图：

**私聊环境**：直接发送即可触发

```
今天广州天气怎么样？
番禺下雨了吗？
明天温度多少？
```

**群聊环境**：需要 @Bot 才会触发

```
@理予 今天广州天气怎么样？
@理予 明天会下雨吗？
```

> 触发关键词：天气、下雨、雨、温度、降温、升温、台风、暴雨、雷暴、刮风、多云、晴、阴天、下雪、雪、雾 等

> 地点提取失败时会自动使用 `nlp_weather_fallback` 配置的默认城市

## 🧠 设计架构

```
用户消息
    │
    ├─ 以 "/" 开头 → 命令路由
    │   ├─ /天气      → cmd_weather()
    │   └─ /天气状态  → cmd_status()
    │
    └─ 自然语言      → on_nlp_weather() 【可选】
        │
        ├─ nlp_weather_enabled=false → 忽略
        ├─ 不含天气关键词           → 忽略
        ├─ 群聊且未@Bot            → 忽略
        └─ 命中关键词 + 符合条件
            ├─ _extract_location_nlp() → 提取地点
            ├─ 未提取到 → 使用 fallback 城市
            ├─ _query_location() → 查询天气
            └─ _build_report() → 回复用户
```

## 🗺 地理编码说明

插件使用 Open-Meteo Geocoding API 进行地点解析，支持以下输入格式：

| 输入 | 解析结果 |
|---|---|
| `北京` | 北京市 |
| `番禺` | 广州市番禺区 |
| `广州番禺` | 广州市番禺区 |
| `上海` | 上海市 |
| `东京` | 东京（日本） |

**地点提取失败时**：自动 fallback 到 `nlp_weather_fallback` 配置项的值（默认"广州"）

## 🔧 常见问题

### Q: 为什么查不到某个城市？

Open-Meteo 的地理编码数据库覆盖全球，但部分小城市/村镇可能无法识别。建议尝试使用上级行政区名称（如"广州市"代替"天河区"）。

### Q: 数据多久更新一次？

实时数据约 15–30 分钟刷新，逐日预报数据每天更新数次。具体取决于 Open-Meteo 的数据同步周期。

### Q: 自然语言触发在群里不生效？

请确认：
1. `nlp_weather_enabled` 已设为 `true`
2. 消息中 **@了 Bot**
3. 消息包含天气关键词（"天气"、"下雨"等）

### Q: 定时推送没有收到？

检查：
1. `reminder_enabled` 是否设为 `true`
2. `reminder_targets` 是否填写了正确的 UMO
3. 使用 `/sid` 获取 UMO 时，需在目标群/私聊中发送
